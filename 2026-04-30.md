## Topic: Integrating SQLModel with Vector Data

Utilized `SQLModel` (which merges SQLAlchemy and Pydantic) to model our database schema.
Created a unified class that acts as both a Pydantic validator and an SQLAlchemy database table definition, including vector columns.
This reduces schema redundancy across backend endpoints and database schemas.

**Code Snippet of the Day:**
```python
from sqlmodel import SQLModel, Field
from pgvector.sqlalchemy import Vector
from sqlalchemy import Column

class DocumentChunk(SQLModel, table=True):
    id: int = Field(default=None, primary_key=True)
    content: str
    embedding: list[float] = Field(sa_column=Column(Vector(1536)))
```
